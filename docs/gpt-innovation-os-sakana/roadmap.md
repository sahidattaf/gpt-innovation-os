Build the system in layers. First create a strong 25-agent operating core, then scale to 50 agents, then to 100+ agents after the operating loop is stable.

## Weekly Review Cadence

Every week, the Controller Agent should answer:

1. What was built?
2. What produced revenue or leads?
3. What failed or got blocked?
4. Which agent performed best?
5. Which workflow needs automation next?
6. What should Claude Code build next?
7. What should be pushed to GitHub?
8. What should be updated in Notion?

## Decision Rule

Only scale to more agents when the current agents have clear ownership, measurable output, and a review loop.

# Contributor Onboarding — Open EU AI Operator

## Welcome
Open EU AI Operator is an open-source operator layer for European SMEs. It helps small businesses adopt AI through practical agents, workflows, templates, and privacy-aware documentation.

## Who This Is For
- SME operators
- AI builders
- Consultants
- Developers
- Notion system designers
- Hospitality, real estate, and marketing professionals

## First Contribution Paths

### Path 1 — Add a Workflow
Use this if you understand a real business process.

Steps:
1. Pick a sector.
2. Define a practical business goal.
3. Write required inputs.
4. Write the workflow steps.
5. Add a human review point.
6. Use fake demo data only.

### Path 2 — Add an Agent
Use this if you want to define a reusable AI assistant.

Steps:
1. Define the agent name.
2. Define purpose and users.
3. Define inputs and outputs.
4. Define guardrails.
5. Add an example prompt.
6. Add success criteria.

### Path 3 — Improve Documentation
Use this if you want to make setup easier.

Steps:
1. Find unclear instructions.
2. Improve clarity.
3. Keep examples practical.
4. Avoid unsupported legal claims.

## Contribution Standards
- Use Markdown.
- Be practical.
- Avoid private data.
- State assumptions.
- Include human review gates.
- Use fake examples.
- Keep legal/compliance language cautious.

## Pull Request Checklist
- [ ] My contribution uses fake or anonymized data.
- [ ] My contribution helps a real SME workflow.
- [ ] I included a human review point.
- [ ] I avoided legal advice claims.
- [ ] I used clear Markdown.
- [ ] I added the file in the right folder.

## Community Rule
Build tools that help local businesses work better, not tools that create confusion, risk, or false confidence.
